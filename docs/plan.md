## Оценка трудозатрат
≈ **8–10 рабочих дней** одного разработчика при разумном темпе. Узкие места:
ER-модель + триггеры (этап 2) и раскатка CRUD на все 20 сущностей (этап 6).

## Точки риска
1. **Триггеры в Postgres на PL/pgSQL** — требуют аккуратной отладки. Митигация: каждый триггер покрыть тестом.
2. **Темизация Qt** — QSS капризен с дочерними виджетами. Митигация: ограничиться 5–7 переменными в токенах и не использовать нестандартные виджеты.
3. **Разрастание дженерик-CRUD** — соблазн делать «универсально для всего». Митигация: жёстко держаться `EntityDescriptor`, спецсцены — только в `ui/queries/`.

## Что НЕ делаем сейчас
- CI/CD pipeline (не требование).
- Электронная упаковка (PyInstaller). Сдаём как `uv run python -m app`.
- Веб-морду к Adminer проксировать никуда не нужно — он сам по себе.
