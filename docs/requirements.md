# Итоговый проект по курсу БД (2 семестр)

Требуется написать приложение, работающее с БД для предметной области заданной для вашего варианта. Приложение может быть как веб, так и Desktop, но не консольное!

Для работы с базой можно использовать:
- Инструменты для выполнения SQL запросов, например, JDBC (но только с использованием prepared statements)
- ORM, например, Hibernate (рекомендуемый мной вариант)
- Что-нибудь ещё, что есть в вашем любимом языке программирования и что я не упомянул (будет любопытно посмотреть)

Язык программирования можно использовать любой на ваше усмотрение. Но крайне рекомендую использовать ООП, SOLID и прочее. «Лапшекод» принимать не буду. Если есть желание можно сделать микросервисную архитектуру, но не обязательно.

СУБД: любая SQL-совместимая. Если придумаете, как можно использовать NoSQL решения – будет плюсом (вариант запихать все таблицы в монгу не засчитывается)

Общие требования: 
1. Ограничения целостности максимально вынести на уровень БД. При необходимости использовать триггеры
2. Хитрые запросы (по варианту)
3. Аутентификация + авторизация
4. CRUD (Create, read, update, delete) по всем сущностям с пагинацией. Подумать над переиспользованием кода
5. Задание данных для подключения к БД в конфиге (IP, логин, пароль, имя БД)
6. Возможность выполнения сырых запросов, заданных пользователем 
7. (Претендующим на 5) Ролевая система вроде ACL. Роли не должны быть прибиты гвоздями, а конфигурироваться суперадмином

Типичный план работ:
1.	Построение модели сущностей (ER-модели)
2.	Реализация приложения
